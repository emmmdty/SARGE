# Paper Experiment Boundaries

This document defines the paper-use boundary for SCI/EI indexed submission
opportunity analysis.

## Allowed

- Use full-train / test-reference evidence as stage experiment material.
- Analyze SCI/EI indexed submission opportunities.
- Position the current package as engineering reproduction, financial event
  extraction system evidence, strict evaluation evidence, controlled study, or
  domain information extraction application.
- Discuss Phase 13 openly as a historical low-score boundary.
- Discuss the S4 full-method test reference as methodology-reference evidence
  only.
- Discuss ProcNet direct-comparable dev results as a baseline table for a
  conservative thesis or SCI/EI indexed opportunity analysis.

## Not Targeted

- CCF-A.
- SCI 1/2.
- Open-source journal strategy.

## Forbidden

- SOTA claim.
- Long-document SOTA claim.
- CCF-A ready claim.
- SCI 1/2 ready claim.
- Treating ProcNet-native reference as the formal main metric.
- Hiding the Phase 13 historical low result.
- Writing S4 full-method test-reference as Phase 13 frozen final result.
- Reinterpreting Phase 13 through later methodology-reference branches.
- Writing evaluator, canonical export, or GETM as algorithmic contributions.
- Continuing to pile regex surface memory or a simple rule planner as the main
  method line.

## Safe Summary

The current package can support a conservative paper-opportunity analysis around
controlled full-train evidence, strict canonical evaluation, and finance-domain
document-level event extraction. It is not a broad leaderboard or top-venue
readiness package.
