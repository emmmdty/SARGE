\section{摘要}

中文金融文档级事件抽取需要在长文本中聚合跨句证据、区分同一文档内的多条同类事件记录，并把每个角色槽位绑定到可评测的文本取值。本文提出 SARGE（Schema-Aware Role-Grounded Extractor），把事件 schema、角色槽位约束与文档候选表面片段组织为受控结构化生成过程：由 Qwen3-4B-Instruct-2507 加 LoRA 适配器在 role-safe JSON 合约下直接生成事件记录，经规则化记录消歧后进入固定槽位评测。本文严格只报告已有完整证据的结果：ChFinAnn-Doc2EDAG \emph{dev}（full-dev，3204 篇）legacy fixed-slot F1 为 85.5，DuEE-Fin 内部 \emph{labeled test}（1171 篇）为 78.0。需要特别说明：上述两数与 EPAL/SEELE 论文公开表格（其在各自 \emph{test} split 上报告）放在同一表格仅作\textbf{公开表格参考点}，而非同测试集 head-to-head；ChFinAnn 与 DuEE-Fin 的主结果分别来自 vLLM BF16 与 HF 4-bit NF4 两个推理后端（详见 §\ref{sec:exp}）。本文进一步用\textbf{已完成的真实运行}给出四类分析：监督微调增益、推理后端对比、解码策略与输出鲁棒性，其中 SARGE 在两数据集上 schema-valid 率为 1.0、零解析失败，DuEE-Fin labeled test 的 exact-record F1（42.8）明显高于 ProcNet 同口径参考（22.7）。本文不声称多种子均值、统计显著性、隐藏测试集/线上 leaderboard 结论或 learned 记录消歧（LRD）的 test 结论，而是把当前可追溯实验证据组织为一个可编译、可核验的中文论文草稿。

\section{相关工作}

\paragraph{文档级事件抽取。}
ChFinAnn 与 Doc2EDAG~确立了中文金融 DEE 的实体—角色填表范式；其后 GIT、DE-PPN、PTPCG、ReDEE、ProCNet 等在图结构、集合预测、剪枝路径与关系增强方向上推进；EPAL 与 SEELE 进一步在跨记录建模与事件结构上取得更强的 fixed-slot 结果。DuEE-Fin 则提供了角色集合更大、同文多事件更密集的中文金融基准。这些工作大多基于专用编码器 + 结构化解码，本文在相同的 fixed-slot 评测口径下，转而以受控 LLM 生成实现端到端抽取。

\paragraph{基于 LLM 的信息抽取与受限解码。}
将抽取重写为生成任务（如 schema 引导生成、function calling）以及在解码阶段施加语法/JSON 约束（grammar-constrained decoding、xgrammar/outlines 等）已较为常见。SARGE 与之的区别不在"用 schema 约束 JSON"本身，而在于面向\textbf{中文金融文档级、多记录}的完整链路：把规则候选作为 surface memory 绑定角色取值、用 anchor 角色对同类多记录做保守消歧、并以严格分离的指标族报告，使生成式系统能与既有 fixed-slot 表格对齐。

\paragraph{中文金融事件抽取。}
中文金融文本数值密集（金额、比例、股数、日期）且实体相邻易混，单纯依赖语言模型记忆容易在相邻数值间出错。SARGE 用规则候选提供可追溯的候选边界，把角色填充变为"从文档证据中选择并组织"，为错误定位提供锚点。

\subsection{任务定义}

给定中文金融文档 $x$ 与事件 schema 集合 $\mathcal{S}$，其中每个事件类型 $t$ 关联一个有序角色集合 $\mathcal{R}_t$。文档级事件抽取的目标是输出事件记录集合 $\hat{\mathcal{E}}=\{(t_i, \{(r, v_{r})\}_{r\in\mathcal{R}_{t_i}})\}$，每条记录给出事件类型与若干"角色—文本取值"对。评测在固定 schema 角色槽位上、同文档同事件类型内进行记录匹配（§\ref{sec:metric}）。SARGE 的核心约束是：$t_i$ 必须取自 $\mathcal{S}$，角色必须取自 $\mathcal{R}_{t_i}$，取值优先绑定到文档中可见的候选片段。

\subsection{Surface memory：规则候选片段}

surface memory 从文档逐段文本用一组确定性正则规则抽取候选表面片段，覆盖中文金融文档的高频取值类型：公司名片段、金额、比例、阿拉伯/中文日期、股份数量、带职务/敬称的人名、引号实体与股票代码（实现见 \texttt{surface\_memory/builder.py}）。每个候选记录其表面串、上下文窗口、来源规则与初步角色分数。生成前按优先级（金额/日期/股数优先）排序并去重，截取至多 40 个候选喂入提示。该机制不声称解决全部数值推理，但为后续角色填充与错误定位提供可追溯的候选边界。

\subsection{Prompt 与 role-safe JSON 合约}

提示按六段拼装：\texttt{[Dataset]}、\texttt{[Schema]}、\texttt{[Document]}、\texttt{[Surface Candidates]}、\texttt{[Event Slot Plan]}、\texttt{[Instruction]}（见 \texttt{generation/prompt.py}）。schema 渲染为"事件类型 $\rightarrow$ 角色列表"逐行文本，并要求模型将角色名\textbf{从 schema 原样复制}，不翻译、不别名、不折叠。输出契约固定为 canonical 形式（见 \texttt{data/canonical.py}）：

\begin{verbatim}
{"doc_id": "...",
 "events": [
   {"event_type": "质押",
    "arguments": {
      "质押方": [{"text": "公司A"}],
      "事件时间": [{"text": "2023-06-01"}],
      "质押股票/股份数量": [{"text": "1000万股"}]}}]}
\end{verbatim}

事件键严格为 \texttt{\{event\_type, arguments\}}，每个角色取值严格为 \texttt{\{text\}} 列表。该契约把"解释性文本混入字段"排除在外，使非法事件类型、非法角色、空预测、解析失败可被\textbf{单独诊断}而非在评测阶段被隐式吞掉。

\subsection{受控生成}

生成阶段使用 Qwen3-4B-Instruct-2507 加 LoRA 适配器。推理以 \texttt{response\_prefix} 为 \verb|{"events":| 引导，主结果采用 $k=1$ 确定性 greedy 解码（\texttt{do\_sample=False}）。系统另支持 schema-aware constrained decoding（SACD，经 vLLM 的 xgrammar guided decoding 将输出约束为按事件类型 \texttt{oneOf} 的 JSON schema），本文将其作为可选诊断而非主结果配置（实现见 \texttt{scripts/infer\_checkpoint.py}、\texttt{scripts/infer\_checkpoint\_vllm.py}、\texttt{generation/schema\_decoding.py}）。

\subsection{解析与诊断}

解析器（\texttt{generation/parser.py}）先做 JSON 抽取（剥离 markdown 围栏、移除前后多余文本），再逐事件校验：事件类型是否属于 $\mathcal{S}$、角色是否属于 $\mathcal{R}_t$、取值形状是否合法，并分别累计 \texttt{parse\_failure}、\texttt{invalid\_event\_type}、\texttt{invalid\_role} 等诊断计数。这些计数在 §\ref{sec:exp} 中作为输出鲁棒性证据报告。

\subsection{规则记录消歧（主路径，no-LRD）}

同类多记录的消歧使用\textbf{规则 planner}（\texttt{postprocess/rule\_planner.py}），也是本文主结果所用路径。每个目标事件类型预定义一组 \emph{anchor 角色}（\texttt{ANCHOR\_ROLES\_BY\_EVENT\_TYPE}，例如"质押"的 anchor 为质押方/质押物/质押股份数量/事件时间）。planner 依次执行：精确去重、空目标事件丢弃、多 anchor 值的保守拆分、anchor 与 arguments 兼容时的保守合并、以及同 anchor 签名下的近重复保留更完整记录。其要点是\textbf{只有 anchor 签名相同且角色兼容的记录才允许合并}，避免把高相似但互斥的真实记录误并为一条。

本文亦实现了一个 learned 记录消歧（LRD）原型（\texttt{postprocess/lrd\_planner.py}，含 safe-anchor 修复，使学习到的 cluster 在合并时继承 anchor/role 兼容边界）。但 LRD 仅作为诊断与未来工作；\textbf{本文主表与全部对比均为 no-LRD 规则路径}，不报告 LRD 的 test 结论。

\subsection{Canonical JSONL 与三轨评测口径}\label{sec:metric}

预测导出为 canonical JSONL（\texttt{evaluation/export.py}），再转评测器格式（\texttt{evaluation/evaluator\_adapter.py}）后由 \texttt{scripts/eval\_three\_tracks.py} 跑三轨。主表只用 \texttt{legacy\_doc2edag\_native\_fixed\_slot}：在同文档、同事件类型内匹配记录，并在固定 schema 角色槽位上统计 $TP$、$FP$、$FN$，
\[
P=\frac{TP}{TP+FP},\quad R=\frac{TP}{TP+FN},\quad F1=\frac{2PR}{P+R}.
\]
该口径用于与 EPAL/SEELE 公开 fixed-slot 表格对齐。指标族边界见表~\ref{tab:metric-families}：\texttt{unified\_strict} 回答 canonical JSONL 的严格文本匹配（并提供 exact-record 诊断），\texttt{docfee\_official} 仅服务 DocFEE official-style 轨道，二者均不与主表混算。

\section{实验}\label{sec:exp}

\subsection{数据集、基线与运行配置}

本文使用两个中文金融文档级事件抽取数据集。ChFinAnn-Doc2EDAG 取其 \emph{dev}（full-dev，3204 篇）作为本轮 ChFinAnn 结果来源；DuEE-Fin 使用内部构造的 \emph{labeled test}（1171 篇）作为主结果来源。\footnote{服务器内部数据集目录名为 \texttt{DuEE-Fin-dev500}，其中 \texttt{dev500} 是数据集构造标签，与文档数无关；该 labeled test split 含 1171 篇文档。} 公开基线采用 EPAL 论文 Table 1 / Appendix Table 4--5 与 SEELE 论文 Table 2--3 的报告值；对原文未报告的 P/R 或单/多事件细分保留缺失值，不做插补。

\textbf{可比性声明（重要）。} EPAL/SEELE 的公开数字是在其各自 \emph{test} split 上报告的，而 SARGE 在 ChFinAnn 为 \emph{dev}、在 DuEE-Fin 为内部 \emph{labeled test}。因此本文所有"高于/低于"均应理解为\textbf{公开表格口径下的参考点比较}，而非同一隐藏测试集或线上 leaderboard 上的 head-to-head。运行配置见表~\ref{tab:run-config}：两个数据集的主结果分别来自 vLLM BF16 merged 与 HF 4-bit NF4 + LoRA 两个后端——这是因为 full-dev 上只完成了 vLLM 整集 run、DuEE labeled test 上只完成了 HF 整集 run（后端差异本身在 §\ref{sec:ablation} 量化）。两结果均来自服务器 \texttt{/data/TJK/DEE/SARGE/} 的完成运行，seed 13。

\subsection{限制}\label{sec:limit}

本文有意保持窄口径：
\begin{itemize}
  \item \textbf{split 可比性}：ChFinAnn 主结果在 dev、DuEE-Fin 在内部 labeled test，与 EPAL/SEELE 的 test split 不同；与公开表格的比较仅为参考点。
  \item \textbf{后端不对称}：两数据集主结果用了不同推理后端（full-dev 仅有 vLLM 整集 run、DuEE labeled test 仅有 HF 整集 run）；ChFinAnn full-dev 的 HF 同后端 cross-check 与 DuEE test 的 vLLM 对照尚未完成。
  \item \textbf{单种子}：仅 seed 13，不报告 mean$\pm$std 与统计显著性。
  \item \textbf{记录消歧}：主结果为 no-LRD 规则路径；learned LRD 仅诊断，不报告 test 结论。
  \item \textbf{无隐藏测试集/leaderboard 结论}：仓库无对应完整证据。
\end{itemize}
这些限制界定了当前证据能支持的表述范围，但不削弱其可复核性：所有数值均绑定到 run root、eval 哈希与本地快照，并由资产生成脚本的完整性断言锁定，防止编辑漂移。

\section{结论}

本文提出 SARGE，一个面向中文金融文档级事件抽取的 schema-aware role-grounded LLM 管线：以规则候选作 surface memory、以 role-safe JSON 合约约束受控生成、以 anchor 规则消歧处理同类多记录，并以严格分离的指标族报告。在公开 fixed-slot 表格参考点意义上，SARGE 在 ChFinAnn full-dev 取得 85.5 F1、在 DuEE-Fin labeled test 取得 78.0 F1，并在两数据集上实现 1.0 的 schema-valid 率与零解析失败。后续工作包括补齐同后端 cross-check 与多种子结果、以及完成 learned 记录消歧的 test 评测。
