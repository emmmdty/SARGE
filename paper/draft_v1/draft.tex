\section{摘要}

中文金融文档级事件抽取需要在长文本中聚合跨句证据，同时区分同一文档内的多条同类事件记录，并把每个角色槽位绑定到可评测的文本取值。本文提出 SARGE（Schema-Aware Role-Grounded Extractor），将事件 schema、角色槽位约束与候选表面片段组织为受控结构化生成过程，由 Qwen3-4B-Instruct-2507 及 LoRA 适配器直接生成 JSON 事件记录，再进入固定槽位评测。本文只报告已有完整证据的结果：ChFinAnn-Doc2EDAG full-dev 3204 篇文档，以及 DuEE-Fin labeled test 1171 篇文档。以 \texttt{legacy\_doc2edag\_native\_fixed\_slot} 为主表口径，SARGE 在 ChFinAnn 上取得 85.49 F1，在 DuEE-Fin labeled test 上取得 77.96 F1。与 EPAL、SEELE 的公开 fixed-slot 表格相比，SARGE 在 ChFinAnn 上达到 85.5 F1，在 DuEE-Fin 上高于 EPAL Table 1 的 76.4 F1，但低于 SEELE Table 3 的 80.8 F1。本文不声称多种子均值、统计显著性、隐藏测试集结果或线上 leaderboard 结论，而是把当前可追溯实验证据组织为一个可编译、可核验的中文论文草稿。

\section{方法论}

\subsection{任务定义}

给定中文金融文档 $x$ 与事件 schema 集合 $\mathcal{S}$，文档级事件抽取的目标是输出事件记录集合 $\hat{\mathcal{E}}$。每条记录包含一个事件类型以及若干角色--文本取值对。金融文档中常见的困难并非只来自单个实体的识别，而是来自跨句证据、多事件共现以及同一角色在多个事件中的重复取值。SARGE 因此不把系统设计为“先自由生成、再事后修补”的流程，而是把输出空间提前收缩到 schema 已定义的事件类型、角色槽位和文档中可见的候选片段之内。

\subsection{Schema-aware role-grounded generation}

SARGE 的核心过程如图~\ref{fig:method-pipeline} 所示。系统首先读取文档文本和数据集 schema，并将事件类型与角色列表渲染为稳定的 schema-slot contract。随后，surface memory 从文档中组织候选表面片段，使生成器在回答每个角色槽位时优先绑定到可追溯文本证据。生成阶段使用 Qwen3-4B-Instruct-2507 与 LoRA 适配器，采用确定性解码，并通过 role-safe JSON 合约约束输出结构。最后，record disambiguation 对同文同类多记录进行整理，避免把多个相近事件误合并为单条记录。

\begin{figure}[htbp]
  \centering
  \includegraphics[width=\textwidth]{method_pipeline.pdf}
  \caption{SARGE 方法流程。方法以 schema-slot contract 与 surface memory 约束结构化生成，并在输出后区分 fixed-slot 主表、unified strict 诊断与 official-style 专用轨道。}
  \label{fig:method-pipeline}
\end{figure}

这一设计的重点是把大模型能力放在结构化决策上，而不是让模型自由改写事件定义。schema 提供事件类型和角色边界，surface memory 提供候选证据，JSON 合约提供可解析输出格式。三者共同约束生成空间，使结果能够直接转换为 canonical JSONL，并进入与历史文档级事件抽取工作相兼容的 fixed-slot 评测。

\subsection{评测口径}

本文主表只使用 \texttt{legacy\_doc2edag\_native\_fixed\_slot}。该指标在同一文档、同一事件类型内进行事件记录匹配，并在固定 schema 角色槽位上统计 TP、FP 与 FN。精确率、召回率和 F1 定义为
\[
P=\frac{TP}{TP+FP},\quad
R=\frac{TP}{TP+FN},\quad
F1=\frac{2PR}{P+R}.
\]
该口径用于与 EPAL Table 1、SEELE Table 2/3 等公开 fixed-slot 表格对齐。内部 \texttt{unified\_strict} 指标回答的是 canonical JSONL 严格文本匹配问题，\texttt{docfee\_official} 只服务于 DocFEE official-style 轨道，二者不与本文主表指标混算。

\section{创新点}

\subsection{Schema-slot 受控生成}

SARGE 将事件 schema 显式写入生成约束，使模型在每个事件类型下只能围绕已定义角色槽位组织答案。相比自由文本生成，这种方式减少了角色名漂移、遗漏槽位和额外虚构字段的风险，也使预测结果可以稳定进入 fixed-slot 评测器。

\subsection{Surface-memory 候选约束}

文档级金融事件通常包含大量公司名、日期、比例和金额，单纯依靠语言模型记忆容易混淆相邻数值。SARGE 将文档中的候选表面片段作为 surface memory 输入模型，使角色填充更接近“从文档证据中选择并组织”的过程。该约束并不声称解决所有数值推理问题，但能为后续错误定位提供可追溯的候选边界。

\subsection{Role-safe JSON 合约}

SARGE 的输出不是开放式自然语言，而是 role-safe JSON 事件记录。固定的 JSON 合约要求模型显式给出事件类型与角色参数，并避免把解释性文本混入预测字段。这个设计使解析失败、空预测、非法事件类型和非法角色能够被单独诊断，而不是在评测阶段被隐式吞掉。

\subsection{记录消歧与指标边界}

同一篇金融文档可能描述多条同类事件，例如多次质押、减持或股份回购。SARGE 在输出后保留记录级边界，并对同类多记录做显式消歧。与此同时，本文把 fixed-slot 主表、unified strict 诊断和 official-style 专用轨道分开报告，避免把回答不同问题的指标压成一个不可解释的综合数。

\subsection{可追溯实验证据}

本文的数值都绑定到具体 run root、manifest hash 和 eval hash。ChFinAnn 主结果来自 full-dev 3204 文档完成运行，DuEE-Fin 主结果来自 labeled test 1171 文档完成运行。DuEE-Fin-dev500 500 文档结果只作为开发诊断补充，不提升为主表性能。这样的证据组织方式使论文草稿可以在不重新训练、不重新推理的前提下被审计。

\section{实验}

\subsection{数据集、基线与运行配置}

本文使用两个中文金融文档级事件抽取数据集。ChFinAnn-Doc2EDAG 保留 3204 篇 full-dev 文档作为本轮报告的 ChFinAnn 结果来源；DuEE-Fin 使用 labeled test 1171 篇文档作为主结果来源。公开基线采用 EPAL 文档中的 Table 1、Appendix Table 4/5，以及 SEELE 文档中的 Table 2/3。对于公开表格未报告的 P/R 或单事件、多事件细分，本文保留缺失值，不做插补。

\input{tables/run_config.tex}

ChFinAnn 完成运行使用 vLLM 0.8.5 与 BF16 merged weights，模型为 Qwen3-4B-Instruct-2507 加 LoRA ep2 merged 权重，seed 为 13，采用 $k=1$ greedy 解码。DuEE-Fin labeled test 完成运行使用 HF Transformers 5.4.0、4-bit NF4 与 LoRA ep2 adapter，seed 同为 13，采用相同的确定性解码设置。两个主结果均来自服务器 `/data/TJK/DEE/SARGE/` 下的完成运行，而不是本地 GPU 推理。

\subsection{主结果}

\input{tables/main_results.tex}

表~\ref{tab:main-results} 给出公开 fixed-slot 口径下的总体结果。SARGE 在 ChFinAnn full-dev 上取得 85.5 F1，高于 EPAL Table 1 的 83.4，也略高于 SEELE Table 2 的 85.1。DuEE-Fin labeled test 上，SARGE 取得 78.0 F1，高于 EPAL Table 1 的 76.4，但低于 SEELE Table 3 的 80.8。这个结论应被理解为当前公开表格口径下的有限比较，而不是隐藏测试集或线上 leaderboard 结论。

从数据集差异看，ChFinAnn 的 schema 边界和角色集合相对更稳定，surface memory 与 schema-slot contract 更容易把候选证据压缩进可控空间。DuEE-Fin 的角色集合更大，同类事件和数值参数更密集，因此 SARGE 虽然超过 EPAL 公开表格，但整体仍低于 SEELE 的公开结果。
