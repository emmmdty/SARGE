## GPU 使用规则

1. **只 kill 自己发起的 PID**，不用 `pkill -f` 或 `fuser -k`（共享服务器，有其他用户）
2. GPU 0 一直是 **Zhhy 用户的进程**，不要碰
3. `CUDA_VISIBLE_DEVICES` 需显式设置，但 QwenBackend 有时会泄漏到其他 GPU
4. 推理前设 `TORCH_COMPILE_DISABLE=1`（否则首次 model.generate() 会编译 5-15 分钟）

## 推理脚本 Bug（关键）

**问题**：`scripts/infer_checkpoint.py` 在 GPU 上卡死。模型加载完成后（日志显示 "generation flags..." 警告），`model.generate()` 调用永远不返回，30+ 分钟无 GPU 进展。CSG/LEEP 阶段正常完成但 GETM 阶段从未写入输出。

**成功的推理路径**：`scripts/train_sft.py` 训练完成后自动推理是成功的（epoch-2 走的就是这条路径，3.4h 完成了 500 docs 推理）。

**推测**：
- epoch-2 推理是在训练完成后立即运行的，CUDA context 已预热
- `infer_checkpoint.py` 独立启动时，首次 `model.generate()` 可能触发 torch.compile 或其他初始化
- `TORCH_COMPILE_DISABLE=1` 设了但可能还不够

**建议修复**：
1. 对比 `train_sft.py` 和 `infer_checkpoint.py` 的 inference 调用的差异
2. 或直接用 `train_sft.py` 的 `--skip-eval false` 路径跑推理（训一个 0-step model 作为 hack）
3. 或在 `qwen_backend.py` 中加 `torch._dynamo.config.disable = True`

```bash

"

# 跑评测
cd /data/TJK/DEE/dee-fin
/home/TJK/.conda/envs/tjk-feg/bin/python -m evaluator <track> \
  --dataset <dataset> --gold <gold.jsonl> --schema <schema.json> \
  --pred <evaluator.jsonl> --out <out.json>
```

可用 track: `legacy-doc2edag`, `unified-strict`, `docfee-official`
